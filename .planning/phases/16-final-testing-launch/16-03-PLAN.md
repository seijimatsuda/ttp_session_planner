<task type="auto">
  <name>Task 1: Test iPad Safari via BrowserStack</name>
  <files></files>
  <action>
This is the CRITICAL test - iPad Safari is the primary target platform for field use.

**Setup BrowserStack:**
1. Open BrowserStack Live (https://live.browserstack.com)
2. Select: iPad Pro 12.9 (latest iOS version available - iOS 17 or 18)
3. Navigate to https://ttp-session-planner.vercel.app

**Authentication Testing:**
1. Navigate to /signup
2. Create account (email: ipad-test+[timestamp]@example.com)
3. Verify signup completes and redirects
4. Logout
5. Login with same credentials
6. Verify session works

**Touch Target Testing:**
1. Check all buttons are easy to tap (44px minimum)
2. Verify form inputs are easy to tap and type
3. Verify links in navigation are tappable
4. Check tag input in drill form is usable with touch

**Video Playback (CRITICAL):**
1. Create a new drill with a video upload (or use existing)
2. Navigate to drill detail page
3. Tap play button on video
4. VERIFY: Video plays inline (NOT fullscreen)
5. VERIFY: No CORS errors in Safari console
6. Drag video scrubber to 25% position
7. VERIFY: Video seeks without reloading
8. Drag video scrubber to 75% position
9. VERIFY: Video seeks without reloading
10. Pause video
11. VERIFY: No errors in console

If video fails:
- Check Safari console for errors
- Note exact error message
- Check if video element has playsInline attribute in inspector
- This is a CRITICAL blocker if fails

**Drag-and-Drop (CRITICAL):**
1. Navigate to /sessions/new
2. Verify drill library sidebar loads
3. Touch and hold drill card for 250ms+
4. Drag drill to grid cell
5. VERIFY: Drag preview follows finger
6. VERIFY: Drill drops into cell
7. Test dragging between cells

If drag fails:
- Check if TouchSensor is activating (may need longer hold)
- Verify click-to-place works as fallback

**Click-to-Place (Fallback):**
1. Tap drill in sidebar (should select it)
2. Tap empty grid cell
3. VERIFY: Drill appears in cell
4. This is the fallback if drag-and-drop is difficult on iPad

**Session Save/Load:**
1. Arrange 2-3 drills in grid
2. Tap save, enter session name
3. VERIFY: Keyboard appears for text input
4. Save session
5. Navigate to /sessions
6. Load saved session
7. VERIFY: Drills populate correctly

Document ALL results, especially any iOS-specific issues.
